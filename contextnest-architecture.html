Licensing Strategy: The Permission System should move to Source Available (non-redistribution) license. This creates commercial lock-in for context delivery while keeping the format and editor extensions open. Anyone can read ContextNest files, but commercial context routing requires our license.

Go-to-Market Strategy

Layer distribution channels, don't choose one. Build developer adoption while pursuing enterprise design partners.

Open Source → Developer Adoption (TipTap extensions, CLI, Format Spec)
↓
MCP Connectors → AI Tool Ecosystem (Claude Code, Cursor, etc.)
↓
Design Partners → Enterprise Requirements (ContextNest + Permission Plane)
↓
Context Market → Monetization Layer (Knowledge marketplace)
↓
Enterprise Platform → Full Commercial (PromptOwl)

Distribution Channels

🟢 Open Source (MIT)

Developer adoption through genuine utility. Build credibility, collect stars, establish presence.

  • @promptowl/tiptap-obsidian — npm package
  • contextnest-spec — Format specification

  • @promptowl/context-parser — Parsing utilities

  • OpenHI tie-in — Potential partner

Context Market / Knowledge Marketplace

The Vision: A marketplace where approved, governed context bundles can be bought and sold. Domain experts monetize their knowledge. Teams buy vetted context sets instead of building from scratch.

How It Works

Actor Action Value
Creator Publishes a Context Set (curated docs on a topic) Earns revenue per download/subscription
Buyer Purchases/subscribes to Context Set Gets vetted, structured context for their AI
Platform (us) Hosts, governs, distributes Platform take on transactions

🌐 OpenHI Integration

OpenHI is building infrastructure for AI-to-AI communication and knowledge exchange. Potential synergy:

  • ContextNest as the format for portable knowledge
  • OpenHI as the network for discovery and exchange
  • Permission Plane as the governance layer for access control
  • Joint marketplace for AI-ready context bundles
